# 開発環境 移行ガイド (Migration Guide)

このプロジェクト（災害報告アプリ）の開発環境を、別のパソコンにコピーしてセットアップする手順です。

## 1. 元のPCでの準備
移行先のPCにファイルを移すための準備をします。以下のいずれかの方法で行います。

### 方法A: フォルダを丸ごとコピー（USBメモリ/OneDriveなど）★推奨★
最も簡単な方法です。ただし、**`node_modules` フォルダはコピー対象から外してください**。サイズが大きく、エラーの原因になりやすいためです。

**コピーするもの:**
- `saigaireport` フォルダの中身すべて
  （ただし `node_modules`, `.git`, `dist` フォルダを除く）

**除外推奨:**
- `node_modules` フォルダ（次の手順で再インストールします）
- `.git` フォルダ（Gitを使わない場合）
- `dist` フォルダ（ビルド生成物）

### 方法B: GitHubを使用（上級者向け）
Gitを使用している場合は、GitHubのリポジトリにプッシュし、新しいPCで `git clone` してください。

---

## 2. 新しいPCのセットアップ
新しいパソコンでプログラムを動かすための準備です。

### 1) Node.js のインストール
1. [Node.js 公式サイト](https://nodejs.org/ja/) にアクセスします。
2. **「推奨版（LTS）」** をダウンロードしてインストールしてください。
3. インストール後、コマンドプロンプト（またはPowerShell）で以下を実行し、バージョンが表示されればOKです。
   ```powershell
   node -v
   npm -v
   ```

### 2) VS Code のインストール（推奨）
[Visual Studio Code](https://code.visualstudio.com/) をインストールしておくと開発がスムーズです。

---

## 3. プロジェクトのコピーと起動
いよいよ新しいPCでアプリを動かします。

### 1) フォルダの配置
準備したファイルを、新しいPCの作業用フォルダ（例: `Documents/saigaireport` など）に配置します。

### 2) 依存関係のインストール
1. VS Codeでそのフォルダを開きます。
2. ターミナル（メニューの「ターミナル」>「新しいターミナル」）を開きます。
3. 以下のコマンドを実行します。
   ```powershell
   npm install
   ```
   ※ これにより、必要なライブラリが自動でダウンロードされ、`node_modules` フォルダが作成されます。

### 3) 開発サーバーの起動
インストールが終わったら、以下のコマンドでアプリを起動します。
```powershell
npm run dev
```
ブラウザで `http://localhost:5173`（または表示されたURL）にアクセスし、アプリが表示されれば完了です！

---

## 4. Google Apps Script (GAS) について
バックエンドのGASはGoogleドライブ上にあります。
- 新しいPCでも、**同じGoogleアカウント**にブラウザでログインすれば、そのまま動作します。
- コードの修正が必要な場合は、GoogleドライブからGASプロジェクトを開いて編集してください。

## 5. 本番公開（Vercelへのデプロイ）
新しいPCからデプロイを行う場合：

1. **Vercel CLI のインストール**（初回のみ）
   ```powershell
   npm install -g vercel
   ```
2. **ログイン**
   ```powershell
   npx vercel login
   ```
3. **デプロイ**
   ```powershell
   npx vercel --prod
   ```
   ※ 初回はプロジェクトとの紐付け（Link）を聞かれる場合があります。画面の指示に従ってYesを選択してください。

## トラブルシューティング

**Q. 'vite' is not recognized... と表示される**
A. `npm install` を忘れていませんか？ またはNode.jsが正しくインストールされていない可能性があります。

**Q. 地図が表示されない / 写真が送れない**
A. インターネット接続を確認してください。また、GASのURLが正しいか確認してください。

**Q. エラーで起動しない**
A. `node_modules` フォルダを一度削除し、再度 `npm install` を実行してみてください。
