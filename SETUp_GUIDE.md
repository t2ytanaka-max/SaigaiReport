# 災害報告アプリ：新規環境セットアップ・移行手順書

本アプリを他の消防団や別のGoogleアカウント環境で利用可能にするための手順を説明します。

---

## 1. 全体構成の確認
このアプリは以下の2つの要素で構成されています。
1.  **バックエンド (Google Apps Script / GAS)**: スプレッドシートをデータベースとして操作します。
2.  **フロントエンド (React App)**: ユーザーが操作する画面（スマホ・PC共通）です。

---

## 2. バックエンド（Google側）の準備

### ① Googleスプレッドシートの作成
1.  新しいGoogleスプレッドシートを作成します。
2.  ブラウザのURLから **スプレッドシートID** をメモします。
    - 例: `https://docs.google.com/spreadsheets/d/【ここがID】/edit`

### ② Google Apps Script (GAS) の作成
1.  スプレッドシートのメニュー [拡張機能] > [Apps Script] を開きます。
2.  プロジェクト名を「災害報告アプリ_Backend」などに変更します。
3.  エディタ上のコードをすべて消去し、プロジェクト内の `backend/code.gs` の内容をすべて貼り付けます。

### ③ IDの設定と初期設定
1.  貼り付けたコードの冒頭にある `const SPREADSHEET_ID = '...';` の部分を、先ほどメモした自分のスプレッドシートIDに書き換えて保存（Ctrl+S）します。
2.  エディタ上部のドロップダウンから **「setup」** を選択し、**[実行]** ボタンを押します。
    - 権限の承認を求められるので、すべて「許可」してください。
    - これにより、スプレッドシートに必要なシート（reports, LiveStreams）が自動作成されます。

### ④ ウェブアプリとして公開
1.  画面右上の **[デプロイ]** > **[新しいデプロイ]** をクリックします。
2.  種類の選択で「ウェブアプリ」を選びます。
    - 次のユーザーとして実行: **自分**
    - アクセスできるユーザー: **全員**（匿名ユーザーも含む）
3.  **[デプロイ]** を押し、発行された **ウェブアプリのURL** をメモします。
    - 例: `https://script.google.com/macros/s/.../exec`

---

## 3. フロントエンド（アプリ画面）の準備

### ① ソースコードの取得と依存関係の解消
1.  本プロジェクトのフォルダ一式（ソースコード）を新しいPCにコピーします。
2.  ターミナル（PowerShellやコマンドプロンプト）を開き、プロジェクトのルートディレクトリで以下を実行します。
    ```bash
    npm install
    ```

### ② 接続先の設定
1.  `src/config.js` を開き、`GAS_URL` を先ほどメモした **GASのウェブアプリURL** に書き換えて保存します。

### ③ アプリのビルド
1.  以下のコマンドを実行し、公開用のファイルを生成します。
    ```bash
    npm run build
    ```
2.  実行後、プロジェクト内に **`dist`** というフォルダが作成されます。これが本番用のファイル群です。

---

## 4. フロントエンドの公開（ホスティング）

生成された `dist` フォルダ内のすべてのファイルを、ウェブサーバーへアップロードします。
主な公開方法の例：
-   **レンタルサーバー**: FTPなどで `dist` の中身をアップロード。
-   **GitHub Pages / Cloudflare Pages**: `dist` フォルダを指定してデプロイ。
-   **Firebase Hosting**: Googleの無料枠などを利用。

---

## 5. 運用開始前の仕上げ

1.  公開したURLをスマホやPCのブラウザで開きます。
2.  もし過去のテストデータが残っている場合は、GASエディタで **「cleanupOldData」** を選択して実行し、シートをきれいにします。
3.  **動作確認**:
    - 通常の報告を送信し、スプレッドシートに反映されるか。
    - LIVE配信を開始し、別端末で視聴できるか。

以上でセットアップは完了です！
